Web Application Security Lab

Hands-on lab environment for practicing web application penetration testing techniques including SQL injection, XSS, and CSRF vulnerabilities.

Web Security Intermediate
GitHub

Lab Overview

This lab provides a comprehensive environment for learning and practicing web application security testing techniques. Designed for intermediate-level security enthusiasts, the lab includes multiple vulnerable web applications that simulate real-world scenarios encountered during penetration testing engagements.

The lab environment is structured to allow progressive learning, starting with basic vulnerabilities and advancing to more complex attack chains. Each vulnerability is intentionally implemented to provide hands-on experience in identifying, exploiting, and remediating security flaws.

Lab Environment

The lab consists of multiple virtual machines and containers configured to simulate a realistic web application infrastructure:

  • Web Server: Apache/Nginx with PHP/Python backend
  • Database: MySQL/MongoDB with intentionally misconfigured permissions
  • Applications: Multiple vulnerable web apps including custom-built and open-source vulnerable applications
  • Network: Simulated corporate network with firewall rules and segmentation

All components are containerized using Docker for easy deployment and reset, allowing learners to experiment freely without concerns about permanently damaging the environment.

Challenges

The lab includes a variety of challenges covering the OWASP Top 10 vulnerabilities:

Injection Vulnerabilities

  • SQL Injection (Error-based, Blind, Union-based)
  • Command Injection
  • LDAP Injection
  • NoSQL Injection

Authentication & Session Management

  • Broken Authentication
  • Session Fixation
  • Insecure Token Generation

Cross-Site Scripting (XSS)

  • Reflected XSS
  • Stored XSS
  • DOM-based XSS

Security Misconfigurations

  • Default Credentials
  • Verbose Error Messages
  • Unnecessary Services

Cross-Site Request Forgery (CSRF)

  • Basic CSRF
  • Token-based Bypass

Advanced Vulnerabilities

  • XXE (XML External Entity)
  • SSRF (Server-Side Request Forgery)
  • File Inclusion (LFI/RFI)

Learning Objectives

Upon completion of this lab, learners will be able to:

1

Identify Vulnerabilities

Recognize various web application vulnerabilities through manual testing and automated scanning

2

Exploit Vulnerabilities

Develop and execute exploits for identified vulnerabilities using appropriate tools and techniques

3

Document Findings

Properly document vulnerabilities with clear steps to reproduce and evidence of exploitation

4

Recommend Remediations

Provide effective remediation strategies for identified vulnerabilities

Tools Used

Burp Suite

Web proxy for intercepting and modifying HTTP requests

SQLMap

Automated SQL injection tool

OWASP ZAP

Web application security scanner

cURL

Command-line tool for making HTTP requests

Results

This lab has been successfully used by over 50 students and security professionals to enhance their web application security skills. Key outcomes include:

  • 95% of participants demonstrated improved vulnerability identification skills
  • 87% successfully exploited at least one high-severity vulnerability
  • 78% showed improved ability to document findings according to industry standards
  • Several participants reported applying these skills in real-world penetration tests

The lab continues to be updated with new challenges and vulnerabilities to reflect emerging threats and attack techniques.
